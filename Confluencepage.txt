Sub-task Title
Add Transverse Risk Flag in Risk Taxonomy Master & Expose via APIs
Description
Introduce a backend-driven mechanism to identify Transverse Risks at L1 taxonomy level without introducing new UI fields. Add a new column IS_TRANSVERSE in WRAP_APPLICABLE_RISK_TYPES, mark Strategic Risk as transverse, and expose this flag through existing select queries and APIs used by Add/Edit Metric screens.
Scope
Add IS_TRANSVERSE column in WRAP_APPLICABLE_RISK_TYPES
Backfill Strategic Risk row with IS_TRANSVERSE = 'Y'
Update select queries to fetch this column along with risk type ID
Update API response models to include transverse flag
Ensure Add/Edit Metric popup receives and displays transverse indication
Out of Scope
No UI structural changes
No new input fields / checkboxes
No changes in metric tables
