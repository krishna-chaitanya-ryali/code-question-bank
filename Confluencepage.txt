Short Description

Production database data corrections to support access flow updates and master data alignment

Description

This database change request is raised to perform controlled, one-time data-level corrections in the production environment to support updated access flow behavior and resolve data inconsistencies.

The change includes:

Corrective updates to access-related records for specific impacted users

Cleanup or adjustment of records in access management tables to ensure proper access lifecycle handling

Required master/dropdown data updates and value corrections to align with current business logic

These changes are data-only in nature and do not involve any schema modifications or application code changes. All updates will be executed using validated SQL scripts during the approved change window.

This is a planned data maintenance activity and not related to any active production incident.

Why is this change being done?

To support corrected access removal and re-provisioning behavior

To resolve data inconsistencies impacting access requests

To ensure data integrity and consistency in production

To avoid repeated manual intervention for access-related issues

Implementation Plan

Obtain approved DB change window and access.

Identify impacted records/users.

Take pre-change data backups (SELECT snapshots).

Execute validated SQL scripts:

Access-related data corrections

Required data cleanup or updates

Master/dropdown value updates (if applicable)

Verify affected row counts.

Commit changes and capture execution evidence.

Business Impact During Implementation

No customer-facing impact

No schema changes

Minimal database load

Validate data restoration
