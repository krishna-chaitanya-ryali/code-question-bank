Short Description
Production database update to inactivate selected locations in RAP application
Description
This change request is raised to inactivate selected locations in the RAP application based on the approved list.
The update ensures that these locations are:
Removed from the RAP menu
Not available in the Access Request form
Not visible in reports (including security reports)
This is a data-only update and does not involve any application code or schema changes.
Reason for Change
To remove inactive locations from the system
To ensure only active locations are available for selection
To maintain data accuracy and consistency
Implementation Plan
Obtain approved change window and DB access.
Identify locations to be inactivated (as per provided list).
Take backup of affected records.
Execute SQL update queries to mark locations as inactive.
Verify updates in the database.
Commit changes.
Customer / Business Impact
No downtime expected
No negative user impact
Improves data accuracy in UI and reports
Technical Impact
No application restart required
No schema changes
Limited to specific location records
Backout Plan
Restore original status using backup data
Revert updated records if required
Post Implementation Verification
Validate locations are inactive in DB
Confirm locations are not visible in:
RAP menu
Access Request form
Reports
Release Plan
Standalone production DB change
Executed during approved window
No application deployment involved
