JIRA Title

Release Task – ERR User Recertification Phase 1 (Dry Run – Limited Users)

Issue Type

Release Task

Environment

UAT → Production

Description

This release is to support a controlled dry run of ERR User Recertification – Phase 1 (Scope 2025) in the production environment.

The dry run will be executed on a limited set of users and line managers only, as identified by the business, to validate end-to-end recertification functionality before full-scale rollout.

Scope of Release

The following activities will be performed as part of this release:

Line Manager Data Preparation

Manually populate Line Manager ID, Name, and Mail ID in USER_TAB

Scope restricted to 2 selected line managers and their direct reports only

Dry-Run Configuration

Temporarily override the standard 31-day recertification logic

Configure system to execute recertification logic on a fixed date (12 Jan) for dry run

Ensure logic is restricted only to selected test users

Dry Run Execution

Execute recertification logic for selected users

Validate:

Access retention / revocation

Correct handling of responses

No impact on non-scope users

Post-Execution Validation

Verify database updates

Confirm access behavior with business users

Capture observations for Go / No-Go decision

Out of Scope

Full-scale recertification for all ERR users

Permanent changes to recertification timelines

Any production data cleanup beyond dry-run scope

Pre-Requisites

Production backup to be taken prior to execution

Confirmation of final user list for dry run from business

Change approval completed

Rollback Plan

Restore database from pre-release backup if any issue is identified

Revert temporary dry-run configuration changes

Risk & Impact

Risk Level: Low

Impact: Limited to selected users only

No impact expected on broader ERR user population

Validation / Sign-off

Business validation to be completed post dry run

Go / No-Go decision scheduled for 12 January
