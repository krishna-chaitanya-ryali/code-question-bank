Database Change Request – Data Correction
Short Description
Production database update to correct demised RAP locations in the Consolidated section.
Description
This change request is raised to perform a data correction in the production database based on a user request.
Some locations that are already demised in the RAP locations data are currently appearing in the Consolidated report section. To maintain data consistency, the relevant records in the database will be updated so that these demised locations are no longer displayed in the Consolidated menu.
This activity involves updating specific records in the relevant tables to ensure the application reflects the correct location status.
These are data-only corrections and do not involve any changes to application code or database schema.
Reason for Change
To align Consolidated section data with RAP location status
To ensure demised locations are not displayed in reports
To maintain accurate and consistent application data
Implementation Plan
Obtain the approved change window and database access.
Log in to the production database server.
Identify the affected location records.
Take a backup of the affected records using SELECT queries.
Execute validated SQL queries to update the relevant records.
Verify the updates in the database.
Commit the changes.
Customer / Business Impact
No customer impact expected
No downtime required
This change improves data accuracy in the Consolidated section
Technical Impact
No application restart required
No database schema changes
Limited to specific location records
Backout Plan
If any issue occurs:
Restore the original records using the pre-change backup.
Revert the updates using rollback queries if required.
Validate the restored data.
Post Implementation Verification
Run verification queries to confirm the updates.
Ensure demised locations are no longer visible in the Consolidated section.
Confirm with the requester after validation.
