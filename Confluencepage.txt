Short Description
Production database update to demise and reactivate selected URL locations
Description
This change request is raised to update the status of selected URL locations in the production database.
The activity includes:
Marking certain URL locations as demised (inactive)
Reactivating selected URL locations as required
These updates are based on approved user inputs and are required to ensure correct visibility and usage of URL locations in the application.
This is a data-only update and does not involve any application code or schema changes.
Reason for Change
To update URL location status as per user request
To ensure only valid locations are active in the system
To maintain accurate and consistent data
Implementation Plan
Obtain approved change window and DB access.
Identify URL locations to be demised/reactivated.
Take backup of existing records.
Execute SQL update queries to modify location status.
Verify the updates.
Commit changes.
Customer / Business Impact
No downtime expected
No negative user impact
Ensures correct location visibility
Technical Impact
No application restart required
No schema changes
Limited to specific records
Backout Plan
Restore original status using backup data
Revert updates if required
Post Implementation Verification
Validate location status in database
Confirm correct visibility in application
Release Plan
Standalone production DB change
Executed during approved window
