8️⃣ How Would You Design a Scalable REST API System?
Answer:
A scalable REST API system should follow a microservices architecture where services are loosely coupled and independently deployable.
We typically place an API gateway or load balancer in front of the services to distribute traffic. Backend services should be stateless so that they can scale horizontally across multiple instances.
For performance optimization, caching mechanisms such as Redis can be used. Message queues like Kafka can handle asynchronous processing for heavy tasks.
Monitoring, logging, and proper error handling are also important to maintain system reliability.
________________________________________
9️⃣ How Would You Scale FastAPI Applications?
Answer:
FastAPI applications can be scaled by running multiple workers using servers like Uvicorn or Gunicorn. The application can be containerized using Docker and deployed on orchestration platforms such as Kubernetes.
Horizontal scaling can be achieved by increasing the number of instances behind a load balancer.
For heavy workloads, asynchronous processing and message queues can be used so that long-running tasks are handled outside the main API request cycle.

🔟 What Is Kafka and Why Is It Used?
Answer:
Kafka is a distributed event streaming platform used for asynchronous communication between services. Instead of tightly coupling services with synchronous API calls, producers publish messages to Kafka topics and consumers process those messages independently.
This architecture improves scalability and reliability.
Kafka is commonly used for event streaming, data pipelines, log aggregation, and microservice communication.

3️⃣ How Would You Design an Event-Driven System?
Example: Kafka use case
Answer:
In an event-driven system, services communicate through events rather than direct API calls.
Explain flow:
1.	Producer
o	Service publishes event to Kafka topic
2.	Kafka Topic
o	Stores events
3.	Consumers
o	Multiple services consume events independently
Advantages:
•	Loose coupling
•	Better scalability
•	Asynchronous processing
Example:
For example, when a new order is created, an event can be published to Kafka and multiple services such as billing, notifications, and analytics can process it independently.

Steps:
1️⃣ Check database queries

2️⃣ Analyze API processing logic

•	Use async processing where possible
3️⃣ Add caching
•	Redis for repeated queries
4️⃣ Use profiling tools
•	Identify heavy operations
5️⃣ Scale the service
•	Add multiple instances behind load balancer
End with:
Performance optimization usually requires improvements across database queries, API logic, and system architecture.
