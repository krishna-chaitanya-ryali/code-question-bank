1️⃣ Project Explanation (Very Important)

If they ask:
“Explain your current project”

You can answer like this:

I am currently working on the Risk Appetite Platform (RAP), which is used to monitor the actual performance of different risk metrics defined in the Risk Appetite Statement (RAS). These metrics are compared against predefined risk appetite and tolerance thresholds.

The platform provides a centralized system for collecting monthly data from multiple business units and storing historical data for trend analysis. Previously this process was handled manually using multiple Excel files and email communication, which was time-consuming and error-prone.

The RAP system automates the entire workflow including data submission, validation, automatic RAG calculation, breach commentary, risk steward validation, and final report generation. The platform also provides dashboards, audit trails, and historical reporting capabilities.

It significantly reduces manual effort, improves data accuracy, and provides better visibility into risk metrics for decision-making.

2️⃣ Process Flow Explanation

You can explain the workflow like this:

The system works in multiple stages:

First, the planner defines the submission dates and sends requests to data submitters.

Data submitters then provide the actual data inputs into the system. Based on the submitted values, the system automatically calculates the RAG (Red-Amber-Green) status using predefined thresholds.

If a metric breaches the threshold, the user must provide breach commentary explaining the cause and corrective action.

After submission, risk stewards validate the data and approve the final submission.

Finally, the system generates reports, including PowerPoint exports, regional reports, and trend charts for the past 12 months.

This explanation matches the process diagram you shared.

The backend of the RAP platform is developed primarily using Python with Flask.

The frontend uses standard web technologies including HTML, CSS, and JavaScript.

The system is hosted on Linux servers and deployed using Jenkins pipelines for continuous integration and deployment.

The web server uses Apache with mod_wsgi to host the Python application.

Your architecture diagram confirms this.

4️⃣ Architecture Explanation

If they ask about architecture:

You can say:

The architecture follows a typical web application model where client requests are sent through the Apache web server.

The Apache server uses mod_wsgi to route HTTP requests to the Python Flask application running in daemon processes.

The backend application processes the business logic, interacts with the Oracle database, and returns responses back to the client through the web server.

This setup allows multiple Python processes to run concurrently, improving scalability and reliability.
